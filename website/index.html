多网卡带宽 聚合加速

HypoMux Plus 把电脑上多条独立网络线路的带宽聚合到同一批高并发下载任务，让 Steam、IDM、浏览器大文件下载跑满每一张网卡。基于 Tauri + Rust 重构，单文件、零依赖、原生般流畅。

Features

为发烧友打磨的每一处细节

从底层网络内核到界面动效，专业、稳定、现代。

双协议无感接管

后台同时运行 SOCKS5 与 HTTP/HTTPS 转发，自动接管系统代理，兼容 Steam、IDM、浏览器。

L3 物理层网卡绑定

IP_UNICAST_IF 接口强绑定，把每条连接物理钉死在指定网卡，根治同网段错网卡问题。

实时遥测大屏

基于内核计数器逐秒采样，合并速度、实时波形、各网卡负载分布一目了然。

全生命周期代理保护

手动停止、启动失败、关窗、退出等所有路径都强制还原系统代理，杜绝代理残留断网。

开机自启 · 自动加速

支持随系统启动、启动即最小化到托盘、开机后自动用上次的网卡开始加速。

现代界面 · 中英双语

克制的工业级控制台设计，深浅双主题，完整简体中文 / English 双语切换。

How It Works

物理多通道并进

加速时程序向系统注册表写入全覆盖代理链条；分流引擎收到连接后，先用 IP_UNICAST_IF 锁死出口物理网卡，再绑定源地址，强制流量剥离默认网关。

多线程下载的连接被 Round-Robin 均匀分发到每张选中的网卡，实现真正的物理带宽叠加。

多线程下载流量
      │  WinINet 系统代理接管
      ▼  http/https → :10801 · socks → :10800
分流引擎 (Rust · tokio)
      │  Round-Robin 轮询分发
      ▼  IP_UNICAST_IF 接口强绑定 + bind
   ├─ 网卡 1 ─┐
   ├─ 网卡 2 ─┼─►  物理带宽叠加
   └─ 网卡 N ─┘
